---

## 📌 프로젝트 개요 (Project Overview)

최근 중고 거래 시장은 친환경 소비와 비용 절감에 대한 관심이 높아지면서 빠르게 성장하고 있습니다. 이에 따라 다양한 중고 거래 플랫폼이 등장했고, 실제 사용자 관점에서 주요 서비스들을 분석해보며 특징과 한계를 파악했습니다.

기존 서비스들은 거래 활성도와 사용자 기반 측면에서는 강점을 가지고 있었지만, 광고 중심의 구조로 인해 사용 흐름이 끊기거나, 단순 거래 기능에 집중되어 사용자 간 상호작용이 제한적인 부분이 있었습니다. 또한 플랫폼 자체의 수익 모델이 광고나 중개에 치우쳐 있어 서비스 확장성 측면에서도 아쉬움이 있다고 판단했습니다.

이러한 점을 개선하기 위해, 본 프로젝트에서는 거래 기능을 중심으로 하되 사용자 경험을 해치지 않는 구조를 우선적으로 고려했습니다. 불필요한 광고를 최소화하고, 사용자 간 소통이 가능한 커뮤니티 기능을 함께 제공하여 단순 거래를 넘어 지속적으로 머무를 수 있는 플랫폼을 지향했습니다. 더 나아가 자체 상품(굿즈) 판매 기능을 도입해 서비스 운영과 확장이 가능한 수익 구조를 함께 설계했습니다.

## 🧪 문제 해결 경험

### 💥 Case: 결제 금액 검증 로직을 통한 결제 신뢰성 강화

#### 🧩 Problem
결제 기능 구현 과정에서 프론트엔드에서 전달되는 결제 금액은  
브라우저 개발자 도구 등을 통해 변경될 수 있다는 점을 확인했습니다.  
이 상태에서 클라이언트가 전달한 금액을 그대로 신뢰하면,  
실제 상품 금액과 다른 값으로 결제가 시도될 수 있는 문제가 있었습니다.

#### 🔎 Cause
- 클라이언트 영역의 값은 사용자가 직접 수정할 수 있음
- 결제 요청 금액에 대한 서버 측 검증이 없으면 위변조에 취약
- 결제 처리 전후의 데이터 흐름을 명확히 관리할 필요가 있었음

#### 🛠 Solution
- 결제 요청 전에 주문 및 결제 정보를 DB에 선 저장하여 기준 데이터를 확보
- 이후 실제 결제 진행 시, DB에 저장된 주문 금액과 외부 결제 API를 통해 전달받은 결제 금액을 비교하는 검증 로직을 구현
- 금액이 일치하는 경우에만 결제를 정상 처리하고, 불일치 시에는 결제를 실패 처리하도록 구성
- 결제 실패 또는 취소 시에는 선 저장된 주문 데이터를 삭제하여 데이터 정합성을 유지

#### ✅ Result
- 프론트엔드 금액 조작 가능성에 대응할 수 있는 서버 검증 구조 확보
- 실제 주문 금액과 결제 금액의 일치 여부를 확인하여 결제 신뢰성 강화
- 결제 실패/취소 상황까지 고려한 데이터 처리 흐름을 구현하며 정합성의 중요성을 체득
